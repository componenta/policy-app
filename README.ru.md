# Componenta Policy App

Интеграционный пакет для `componenta/policy` в Componenta-приложении. Он превращает атрибуты политик доступа в сериализуемые описания, которые `componenta/policy` может загрузить из продакшен-кеша.

Если коду нужно только проверять права, он должен зависеть от `componenta/policy`. Этот пакет нужен приложению во время сборки кеша.

## Установка

```bash
composer require componenta/policy-app
```

Пакет объявляет `Componenta\Policy\App\ConfigProvider` в `extra.componenta.config-providers`.
Если установлен `componenta/composer-plugin`, провайдер автоматически добавляется в сгенерированный список провайдеров.

## Связанные пакеты

| Пакет | Зачем нужен здесь |
|---|---|
| `componenta/policy` | Выполняет проверку прав и читает скомпилированные описания политик. |
| `componenta/class-finder` | Находит классы и публичные методы с атрибутами политик. |
| `componenta/app` | Запускает компилятор только когда policy-пакет установлен и связан в контейнере. |

## Что добавляет

Пакет предоставляет `Componenta\Policy\App\Compile\PolicyMapCompiler`.

Компилятор сканирует найденные классы на атрибуты политик:

- на классах;
- на публичных методах.

Он возвращает карту описаний, где ключом является id действия. `CompiledPolicyProvider` из `componenta/policy` использует эту карту во время выполнения.

## Режим разработки

В разработке метаданные политик могут читаться через рефлексию или пересобираться во время прогрева кеша. Это удобно, пока классы и атрибуты меняются.

## Продакшен

В продакшене приложение должно загружать скомпилированные описания вместо сканирования исходников. Так старт запроса остаётся предсказуемым и не выполняет повторную рефлексию по классам команд, запросов и контроллеров.

Компилятор должен включаться только когда `componenta/policy` установлен и провайдер политик связан в контейнере. `componenta/app` выполняет эту проверку через поддержку компиляции.

## Ошибки

Проверка прав остаётся в `componenta/policy`. Если скомпилированное описание устарело или некорректно, рантайм-провайдер политик обрабатывает это как ошибку политики. App-пакет только создаёт карту описаний.

## Границы

`componenta/policy-app` не принимает решений об авторизации. Интерфейсы политик, атрибуты, провайдеры, композиция политик и enforcement принадлежат `componenta/policy`; этот пакет отвечает только за компиляцию для приложения.
